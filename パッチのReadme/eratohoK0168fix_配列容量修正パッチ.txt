◆eratohoK Ver0.168fix用 乱数配列拡張パッチ

・概要
　ランダムに順番を決める関数で使用する配列の長さが少し足りないようなので拡張しました
　これで特賞当選率0.1%の引き切りガチャが作れるよ、やったね

　あと、なんとなくあると便利そうな「勢力内で特定の能力について最高値を持つキャラを探す」関数も作ってみた

・パッチ内容
＋VARIABLE.ERH
　SHAFFLE_ARRAYのサイズを1000に

＋SORT.ERB
　@FISHER_YATES_SHAFFLE
　例外によるエラー落ちを検出する上限を引き上げ

＋SLG_FUNCTION.ERB
　@CALC_MOST_POWER_C
　[第一引数]勢力所属キャラのうち能力値[第二引数]が最も高いキャラクタ番号を返す関数
　[第三引数]に1を入れた場合、更に未行動状態でないキャラも弾く
　なお、第二引数はGETNUM関数を使うと多少見やすくなるかも

・導入方法
　Ver0.168fixに上書きしてご利用ください。

・補記
　ランダムキャラが口上デイリー持てないのもどうかと思うので、そちらから封じるのはちょっとやめておきます
　というより、CSVにない固定キャラを足してキャラ数を計算するのがすごく面倒な気配がしたのでやめておきます

　他のところでも500オーバーの乱数配列が必要な物を何か作れればなぁ。
