eratohoK 0.251用 データ操作に機能追加 修正

12[複数キャラクターをランダムで死亡させる]
13[ランダムキャラ複数追加]
14[カスタムキャラのクローンを複数生成]

を追加してみた。

イベントを追加するパッチとか作るマンの動作確認時に役立つかもしれないし、役に立たないかもしれない。

まあ、自国にランダムキャラ士官をまとめて追加するチート手段として使ってもいいし。
ランダムキャラを使用しない選択にしたけど、やっぱり少し欲しくなった場合に使って放浪させてもいいし。


既存のファイルやセーブデータを別の場所に保存してから、
ERB\SYSTEM\CONFIG_DATA_EDIT.ERB
に上書きで。

githubのdevelopブランチから取得して遊んでる人は
CONFIG_DATA_EDIT.ERB.for_github_developブランチ
をリネームして使ってね。こっちでは動作確認してないけど。

変な不具合あったらスマン。
１つ発見したから修正したけど。



不具合修正履歴
2017/03/20
・ランダムキャラが既に満員の場合、14[カスタムキャラのクローンを複数生成]に入るとおかしくなるのを修正（入ってすぐ出るようにした）
（カスタムキャラの情報を見せるためにADDVOIDCHARAしてランダムキャラ仕様に情報を書き換えて用が済んだらDELETE_CHARAするけど、既に満員の場合にはNO:対象がランダムキャラの範囲を超えてしまってFLAG:汎用武将カウントが狂っていた）

仕様
・特殊キャラ扱いを5000人追加とかやったらエラーの原因になる。
ランダムキャラは途中で止まるので大丈夫のはずだけど、データ操作以外などの箇所に不具合があったらその限りではない。
・13[ランダムキャラ複数追加]
で、ランダムキャラが満員の時には特殊キャラ扱いで追加することもできない
（ランダムキャラを1人生成してから特殊キャラ仕様に書き換えているので。）

機能を用意してもいいかもしれないけど実装してない
・指定した１キャラをDELETE_CHARAする
・"ランダムキャラ"と"特殊キャラ扱い"だけではなく、"子供キャラ扱い"でも追加できるようにする。
