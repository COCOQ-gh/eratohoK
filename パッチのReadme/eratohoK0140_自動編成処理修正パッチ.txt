∇eratohoK Ver0.140用 自動編成処理修正パッチ

▲概要
AIが部隊を編成する時に使用する能力の比重値（AI_PRIORITY:X:Y）を参照するだけでなく直接操作していたのを修正
あと、直接倍率型の計算式をちょっと変更（評価値が細かく変化するように）

■パッチ内容
・SLG_AI.ERB
　@CHECK_COMMANDER_BEST
　関数内でAI_PRIORITYを参照するための配列FREE_AI_PRIORITYをセットし、そちらを使用するように

　比重値にゆらぎを発生させるために20%程度のズレが起こるように乱数をセットしていましたが、
　比重値を上書きした上であとの処理で用いているため、呼び出されるたびにズレが発生するようになっていました

・SLG_FUNCTION.ERB
　@ABL_POWER_X
　評価値の計算式を以下に変更（通常の時点で）、また最低１を返すように
　能力値のみの評価値 * SQRT(感情補正値 * オプション補正 * 10000 / 30) / 1000

　補足：最後の割り算をSQRT（平方根）内に入れるとSQRT内は
　　　　感情補正値 * オプション補正 * 10000 / 30 / (1000 * 1000)
　　　＝感情補正値 * オプション補正 * (10 * 1000) / (30 * 1000 * 1000 )
　　　＝感情補正値 * オプション補正 / (30 * 100)
　　　＝感情補正値 * オプション補正 / 3000
　　　　となり元と変わりはありません（SQRT関数の切り捨てで多少変化はしますが）


▲パッチ適用方法
eratohoK 0.140本体にこのパッチを上書きしてください。

〓その他
１つ目の方で済むかと思っていたけどこっちの値は分子側なので、
分母側でないと発生しない0除算とは関係なかったのに気づいた時かなり焦ったのは秘密です

どっちも自分が過去に作ったパッチ由来なのが悲しいところ。
