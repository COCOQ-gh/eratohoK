済マークパッチ＋α

特定条件化で夜這いに君主が表示されるバグを修正
捕虜の扱い画面を多少変更（性別表示の追加、名前部分の横幅拡張）
そのターンに会っていることが理由で選択できないキャラを「済」マーク付きでグレー表示するように（オプションで以前の設定に戻せます）


以下開発者向け

性別ごとの色定数を追加

メニューの選択可否判定をコンパクトに

@PRINT_PARTNER_DATAのARG:4を仕様変更、@CHECK_SELECTABLEの返り値をそのまま投げていいように
CHECK_SETECTABLE{n}の返り値について
0:通常の選択不可（リストに載らない）
1:選択可能
2:灰色でリストに載るが選択不可能
3:灰色でリストに載り選択（確認）可能だが実際にコマンド実行はできない（確認画面側の変更も必要、ただし3を返すものは現状未実装）
0より大きい値を返すキャラが1人もいなければコマンドは選択不可能に
負値は何らかの用途ができたときに使う形で
あくまでSELECTABLEということで表示に関わる判定のみ残すようにしています
表示は変わらないけど実行時の分岐に関わるものは実行時に判定でいいと思います
両方で使う判定は個別に関数化という形で
論理演算子の返り値で非1のTRUEが返ってくることがないのかちょっと不安

@PRINT_SEXと@SETCOLORBYSEXを追加
既存の一部性別表示部分を上記関数に差し替えてコードを多少コンパクトに

他にもファイルの分離や共通コードの関数化などいくつか

全体として重くなる（ループ回数が増える等）可能性がある変更でも、プレイ上問題ないと思われるものは保守性を優先して書き直しています。
化石PC利用者等で重い人が出てきたら何か考えるかも。
もちろん軽くしてる部分もありますよ！（どちらにしても一部以外は速度なんて誤差だとは思いますが）
