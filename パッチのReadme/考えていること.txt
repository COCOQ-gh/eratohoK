・デイリーイベント派生のデイリーイベント
　あるデイリーイベントをトリガーとした別のデイリーイベントの誘発、イメージとしては
　＋恩返しイベントで通い妻ゲットしてイチャコラ
　→エロいことにどっぷりハマった娘の惚気話を聞いた周囲が興味を持つ
　→発情期に娘と一緒にやってくるので、親子丼なり姉妹丼なりおいしくいただく
　→噂が噂を呼びとうとう里にお呼び出しされ、そのままいろいろごちそうさまでした

　といったところ、各ステップのハードルを高めにして最終的な報酬もそれなりにしてみるのはどうかなとも思った次第
　報酬例:不定期な兵力増強（産めよ増やせよ地に満ちよ）、素質獲得（抜け道を教えてもらい疾風迅雷など）
　あと、毎回変数を文字列変換するぐらいならデイリー用の文字列配列1つ用意してそれを使うのはいかがでしょうか？
　周回ごとにお気に入りの遊女と遊ぶイベントとか、募金娘が名前を持つとか、薬売りの屋号を決めるとか、「越後屋、お主も悪よのぉ」の屋号をいろいろ変えるとか、おもしろそうじゃない？

・AIの自動進行の細分化
　スレで色々あった話で一番興味を引いたので実装に向けて下調べしていたところ、いくつか問題がわかったのでまとめておく
　とりあえず細分化できそうなのは以下のとおり
　＋士官の募集
　　結構厄介で、手動の場合現状拠点フェイズ1ターンと引き換えに実行できる状態です
　　AI側では人数による上限が付いている（勢力内人数が8人以下なら雇用する）ので、それとの兼ね合いも必要と思います
　　（手動だと勢力の人数に応じて複数回行動か国庫から金を出させるか、かな？）
　＋外交
　　AI側の会談アクションに費用の概念を入れ忘れているのでそれを入れた上で、
　　プレイヤー側も国庫から拠出させていく方向になると思います
　　後はそんなに気にはしなくてもいいのかも
　＋政策
　　ここは特に弄る予定は無し
　　政策システムそのものの仕様変更は一応構想中（某SMAC型の複数の分野から1つづつ選んでそれぞれが各方面に補正を掛けるイメージ）
　＋攻撃されている都市への救援部隊の作成
　　編成の自動化に関するオプションになると思います
　＋徴兵
　　私費による徴兵は残す（代わりに効率を落とす）方針で考えています
　＋編成及び侵攻
　　残念ながら現行のシステムではコレを１つの塊で処理しないとややこしいことになるのでちょっと保留
　　（侵攻先を決定した後の編成タイミングで敵がいない自勢力の都市にいる部隊を侵攻部隊にできるようにすれば分割は可能）
　＋防衛部隊の分配
　　ここは問題無いでしょう
　　むしろ配分の計算処理を少し見なおせという話もあるかもしれず

　コレぐらいに分けておいてそれぞれオン・オフするという形式
　ただし、戦略フェイズ画面での自動化をしなければ全て手動で動くという想定

・国庫と所持金のすみ分け
　とりあえずお金を使っているところを洗い出して、国庫に回せるのは国庫にまわしていければなぁと思ってます
　分かりやすい基準としては「AI側の行動でお金を使わせたいところを国庫消費に回し、自勢力側での操作もそれに合わせる」というところ
