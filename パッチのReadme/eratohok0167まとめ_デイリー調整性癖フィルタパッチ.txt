◆パッチ内容
・EVENT_DAILY.ERB
　@EVENT_DAILY
　強制イベント及び共通デイリー、口上デイリーの３段階で処理を行うように
　共通デイリー及び口上デイリーの発生順をランダムに
　共通デイリーについては発生時期／発生確率をこちらで管理するように
　補足：当初予定していた「複数発生の回避」は確率を見て要らないと判断しました
　イベント数上限を設けるなら発生確率も引き上げるかなぁ（現状でも全部発生するとして何もない確率が大体5割切っている）
　口上イベントについては毎ターン最大3人の制限を追加（通常終了したイベント（RETURN 0を返したもの）を1回とカウント）

▲パッチ適用方法
eratohoK ver0.167本体にパッチまとめ改を適用後、このパッチを上書きしてご利用ください

何分一人でチェックしたものなので見落とし打ち間違いなど細かい不具合があると思いますので、
気になったところは報告いただければ幸いです

〓その他
フィルタ設定基準について
・あなたの視界外である
・あなたに対する行為がネガティブに傾く
・勝手に性に関する経験が増える
・一定レベルの選択による回避が不可能
といったところに挟んでいます
なので、夜遊びは（選択肢を消すのが面倒なのもあるけど）選択可能なのでフィルタが入っていませんし、
富豪などは選択肢前の文章などでいろいろ示唆してるので、まるごとフィルタをかけています

また、フィルタは基本的に呼び出すところではなく関数側でチェックさせています
（こうしておかないとその関数を別のところで呼び出すときにそこにもフィルタチェックを掛ける必要があるので）

あくまでも独断と偏見と気まぐれと技術の及ぶ範囲でやっております
